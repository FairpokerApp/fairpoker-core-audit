页面上有两个不同的指纹（CID），它们本来就不一样、也永远不该相等：

  • 牌局客户端 CID = 编译好、能直接玩的成品（好比烤好的蛋糕）。
  • 核心源码包 CID = 给人阅读、审查的源代码（好比面粉和鸡蛋）。

面粉的编号和蛋糕的编号永远不会一样——这很正常。你不要拿这两个互相比。 正确做法是各自验证：

  1. 比对 ①：你下载的源码包算出的 CID == 网站公布的核心源码包 CID → 证明你读的源码就是官方那份，没被掉包。
  2. 比对 ②：你用这份源码重新编译出的成品 CID == 网站公布的牌局客户端 CID → 证明线上跑的游戏，正是这份源码编出来的。

① + ② 合起来 = 你能读的（源码）就是线上真在跑的（游戏）。 老板没法「给你看一份干净代码，背地里却跑一份作弊代码」——因为任何人重编都会当场对不上、立刻穿帮。

The site shows two different fingerprints (CIDs), and they are not the same and never should be:

  • Game client CID = the compiled, ready-to-play build (the baked cake).
  • Core source package CID = the readable source code you audit (the flour and eggs).

A bag of flour and a cake never share a label — that's normal. You do not compare these two to each other. Instead you verify each one independently:

  1. Check ①: the CID of the source package you downloaded == the published Core source package CID → proves the source you're reading is the official one, not swapped.
  2. Check ②: the CID of the client you rebuilt from that source == the published Game client CID → proves the live game is compiled from exactly that source.

① + ② together = what you can read (source) is what is actually running (game). The operator cannot show you clean code while secretly running cheating code — any independent rebuild would fail to match and expose it instantly.

サイトには2つの異なる指紋（CID）があり、これらは別物で、一致することは決してありません：

  • ゲームクライアント CID = コンパイル済みで遊べる成果物（焼き上がったケーキ）。
  • コアソースパッケージ CID = 監査用の読めるソースコード（小麦粉と卵）。

小麦粉とケーキの番号が一致しないのは当然です。この2つを互いに比べてはいけません。 それぞれ独立に検証します：

  1. 照合 ①：ダウンロードしたソースパッケージの CID == 公開のコアソースパッケージ CID → 読んでいるソースが公式で、すり替えられていない証明。
  2. 照合 ②：そのソースから再ビルドした成果物の CID == 公開のゲームクライアント CID → 稼働中のゲームがまさにそのソースから作られた証明。

① + ② = 読めるもの（ソース）が実際に動いているもの（ゲーム）と同じ。 運営が「きれいなコードを見せて、裏で不正コードを動かす」ことはできません。誰が再ビルドしても一致せず、即座に露見します。

El sitio muestra dos huellas (CID) distintas, y no son iguales ni deben serlo:

  • CID del cliente de juego = la compilación lista para jugar (la tarta horneada).
  • CID del paquete de código = el código fuente legible que auditas (la harina y los huevos).

Una bolsa de harina y una tarta nunca comparten etiqueta — es normal. No compares estos dos entre sí. Verifica cada uno por separado:

  1. Comparación ①: el CID del paquete que descargaste == el CID del paquete de código publicado → prueba que el código que lees es el oficial, no sustituido.
  2. Comparación ②: el CID del cliente que reconstruiste == el CID del cliente de juego publicado → prueba que el juego en vivo se compiló exactamente desde ese código.

① + ② juntos = lo que puedes leer (código) es lo que realmente se ejecuta (juego). El operador no puede mostrar código limpio y ejecutar en secreto código tramposo: cualquier reconstrucción independiente no coincidiría y lo delataría al instante.

Le site affiche deux empreintes (CID) différentes, qui ne sont pas identiques et ne doivent jamais l'être :

  • CID du client de jeu = la build compilée et jouable (le gâteau cuit).
  • CID du paquet source = le code source lisible que vous auditez (la farine et les œufs).

Un sac de farine et un gâteau ne partagent jamais d'étiquette — c'est normal. Ne comparez pas ces deux-là entre eux. Vérifiez chacun séparément :

  1. Comparaison ① : le CID du paquet source téléchargé == le CID du paquet source publié → prouve que le code que vous lisez est l'officiel, non remplacé.
  2. Comparaison ② : le CID du client que vous avez reconstruit == le CID du client de jeu publié → prouve que le jeu en ligne est compilé exactement à partir de ce code.

① + ② ensemble = ce que vous pouvez lire (source) est ce qui tourne réellement (jeu). L'opérateur ne peut pas montrer du code propre tout en exécutant secrètement du code tricheur — toute reconstruction indépendante ne correspondrait pas et le révélerait aussitôt.

Die Seite zeigt zwei verschiedene Fingerabdrücke (CIDs), die nicht gleich sind und es nie sein sollten:

  • Game-Client-CID = der kompilierte, spielbare Build (der gebackene Kuchen).
  • Kern-Quellpaket-CID = der lesbare Quellcode, den du prüfst (das Mehl und die Eier).

Ein Sack Mehl und ein Kuchen teilen nie dasselbe Etikett — das ist normal. Vergleiche diese beiden nicht miteinander. Prüfe jedes einzeln:

  1. Vergleich ①: die CID des heruntergeladenen Quellpakets == die veröffentlichte Kern-Quellpaket-CID → beweist, dass der gelesene Quellcode der offizielle ist, nicht ausgetauscht.
  2. Vergleich ②: die CID des neu gebauten Clients == die veröffentlichte Game-Client-CID → beweist, dass das Live-Spiel genau aus diesem Quellcode kompiliert wurde.

① + ② zusammen = was du lesen kannst (Quellcode) ist das, was tatsächlich läuft (Spiel). Der Betreiber kann nicht sauberen Code zeigen und heimlich betrügerischen Code ausführen — jeder unabhängige Neubau würde nicht übereinstimmen und es sofort aufdecken.
